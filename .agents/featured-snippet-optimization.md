- **Conduct regular privacy audits** – Review data collection practices, vendor compliance, and security measures quarterly to identify and fix gaps

For each optimized post, ensure:

- [ ] H2 heading includes target keyword
- [ ] Proper HTML tags used (h2, ul, ol, table)

### On-Page SEO
- [ ] Primary keyword in H2 heading
- [ ] Primary keyword in first paragraph
- [ ] Schema markup added where appropriate:
  - HowTo schema for numbered lists
  - FAQPage schema for Q&A sections
  - Article schema for all posts
- [ ] Alt text on any images in snippet section
- [ ] Internal links to related posts

### Format Requirements by Type

**Definition Boxes:**
- 40-60 words exactly
- Bold the term being defined
- Format: Definition + How it works + Why it matters
- Primary keyword in first 3-5 words

**How-To Lists:**
- 5-8 numbered steps
- Each step is 1-2 sentences
- Start with action verb
- Progressive order (step 1 → step 2 → etc.)
- Use `<ol>` HTML tag

**Table Snippets:**
- 3-5 columns (readable width)
- 4-6 rows of data (plus header)
- Clear column headers
- Concise cell content (under 20 words)
- Use proper `<table>`, `<thead>`, `<tbody>` tags
- Bold row labels in first column

**List Snippets:**
- 5-10 bulleted items
- Each item 1 sentence
- Bold the key term/phrase
- Parallel structure across items
- Use `<ul>` HTML tag

## Common Pitfall Avoidance

### What NOT to Do

1. **Don't keyword stuff** - Snippets should read naturally
2. **Don't make answers too long** - Respect word/item count limits
3. **Don't use complex language** - Simple, clear answers win
4. **Don't bury the answer** - Place snippet-worthy content early
5. **Don't ignore mobile** - Test snippet readability on mobile devices
6. **Don't forget to update** - Stale content loses snippets
7. **Don't optimize for multiple snippets per page** - Focus on one primary snippet per post
8. **Don't ignore search intent** - Answer exactly what the query asks
